# Core Domain Module

This module defines the fundamental entities and data structures that represent the core concepts and business logic of the application.
It establishes the vocabulary and structure for the system's internal state.

## File Structure

- `agent_capabilities.py`: Defines the capabilities and functionalities an agent can possess, such as the tools it can use or specific actions it can perform.

- `agent_identity.py`: Contains data models for representing the unique identity and characteristics of an agent.

- `conversation_context.py`: Defines the structure for storing and managing the context of ongoing conversations with an agent.

- `cross_agent_context.py`: Describes the context information that can be shared or understood across multiple agents.

- `registered_agent.py`: Represents the data structure for an agent that has been registered within the system, including its configuration and state.
